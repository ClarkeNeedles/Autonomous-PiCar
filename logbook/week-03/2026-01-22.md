# First CV Model: Duck Detection

> **Session Date**: January 22, 2026
> **Duration**: 10:00 AM - 1:00 PM (3 hours)
> **Location**: BMH second floor

## 🎯 Objectives

Create and validate the first custom computer vision model to detect ducks.

- Collect videos of ducks to segment into images.
- Annotate data using Roboflow.
- Train YOLOv8n (Nano) model.
- Validate inference performance on PC.

## 📋 Detailed Work Log

### Session 1: Data Collection & Annotation (10:00 AM - 11:00 AM)

**Description**:
Created the initial dataset for the target object.

**Process**:
1. Took videos of the duck to train model.
2. Uploaded images to Roboflow for annotation.
3. Drew bounding boxes around the ducks in all images.
4. Applied augmentation (flip, rotate, brightness) to increase dataset size to 150 images.

### Session 2: Model Training (11:00 AM - 12:00 PM)

**Description**:
Trained the YOLOv8 model using Google Colab GPUs.

**Process**:
1. Exported dataset from Roboflow in YOLOv8 format.
2. Set up a Google Colab notebook with GPU acceleration.
3. Trained the 'yolov8n' (nano) model for 50 epochs.
4. Downloaded the resulting "best.pt" weight file.

## 🐛 Challenges & Solutions

### Challenge 1: False Positives

**Problem**: The model initially detected yellow shapes notes as ducks.
**Analysis**: The dataset contained only yellow ducks on clean backgrounds, so the model overfit on the color yellow rather than the shape, future models will contain more images with varying conditions.
**Solution**: Add "null" background images (photos of the lab without the duck) to the training set to teach the model what is NOT a duck.
